# Twick editor final review fixes report

## Summary

Implemented all 2 Critical + 6 Important final-review fixes for the Twick editor integration branch.

## Fixes

### Critical 1: Server-render SSRF guard

- Save API now rejects raw URL media sources in media fields before persisting `projectData`.
- Render worker recursively resolves only `mediaobj://` media references through `resolveMediaUrlForServerRender` with tenant/project/editor/episode context.
- Non-`mediaobj://` values in media fields (`src`, `*Src`, `*Url`, `poster`, etc.) with URL schemes such as `http:`, `https:`, `file:`, or `data:` throw `EDITOR_RENDER_INVALID_MEDIA_SOURCE` before reaching `@twick/render-server`.
- Ordinary non-media strings remain untouched, so subtitle/text fields are not blocked merely because they contain URL-like text.
- `resolveMediaUrlForServerRender` now refuses non-`mediaobj://` refs directly.

### Critical 2: Render DONE before billing settlement

- Render worker computes actual render minutes from timeline duration before rendering and enforces `actual <= frozen/quoted` (`EDITOR_RENDER_BILLING_FREEZE_UNDERESTIMATED`) before any render/upload/DONE persistence.
- With the hard gate, normal settlement is covered by the pre-freeze quantity.
- Worker lifecycle catches render settlement failures and rolls back `renderStatus=DONE` + `renderOutputMediaObjectId` by `renderTaskId` before failing the task.
- General terminal failure cleanup also clears DONE render outputs by `renderTaskId`, so a render artifact cannot remain DONE after unpaid/failed settlement.

### Important 1: Caption billing before persistence

- Caption handler parses frozen/quoted minutes from task payload.
- Caption build now calculates actual minutes before DB update and throws `CAPTION_BILLING_FREEZE_UNDERESTIMATED` if actual exceeds frozen quantity.
- Timeline is not persisted when the hard gate fails.

### Important 2: smart_crop billed seconds

- `smart_crop` now returns `actualSeconds` / `actualQuantity` as `ceil(processed segment seconds)` instead of `0`.
- Existing pre-freeze from the route remains based on selected video duration, and worker enforces `actual <= frozen`.
- Pricing item already exists for `EDITOR_AI_ENHANCE_SMART_CROP`; settlement now receives non-zero seconds.

### Important 3: smart-cut version CAS

- Smart-cut persistence now uses `updateMany({ where: { id, version } })`.
- On version conflict, it reloads latest editor project data and rebuilds/merges before retrying, up to 3 attempts.
- Latest non-generated user tracks are preserved while generated video/audio tracks are replaced.

### Important 4: render retry/watchdog lock handling

- Render handler only writes `renderStatus=FAILED` on the final non-retry attempt; retryable failures keep the PROCESSING lock for BullMQ retry.
- Task service releases render locks by `renderTaskId` when tasks fail, cancel, or are swept as stale by watchdog.
- Queued cancel resets render state to IDLE; processing cancel/failure/watchdog marks FAILED without touching newer task bindings.

### Important 5: voice duration unit

- Frontend editor-stage voice mapping now treats `voiceLine.audioDuration` as milliseconds via `durationMsToSeconds`.
- Added regression coverage for `audioDuration=5000 -> duration=5`.

### Important 6: voice-optimize dedupe key

- Voice optimize dedupe key now includes `voiceLineId`.
- It no longer depends on optional `selectedElementId`, preventing cross-voice-line false dedupe for identical content/speaker/speed.

## Impact on other functionality

- Media validation is scoped to media fields only; regular text/subtitles are preserved.
- Render retry behavior is safer: retryable failures remain PROCESSING for the same task; terminal paths release the lock by task id.
- Smart-cut now preserves unrelated user-added tracks while refreshing generated smart-cut tracks.
- Known unrelated typecheck errors remain in `tests/unit/components/art-style-library/ArtStyleEditor.test.tsx` around missing `@testing-library/react` / jest-dom matcher types.

## 修复轮次 2

### Critical 1: SSRF media source allowlist

- PUT 保存层和 render worker 都从“只拒绝带 URL scheme 的字符串”改为媒体字段白名单：媒体字段中的非空字符串必须是 `mediaobj://<non-empty-id>`。
- 媒体字段识别限定在 Twick 媒体元素（`type=video/audio/image`）本体字段，以及元素 `props` 下的 `src` / `source` / `url` / `poster` / `*Src` / `*Url` 等资源字段，避免误伤字幕文本、`props.label`、`metadata.source='generated'` 等普通字符串。
- render worker 在调用 `@twick/render-server` 前只解析 `mediaobj://` 为服务端签名 URL；`//169.254.169.254/...`、`//127.0.0.1/...`、`/etc/passwd`、`relative.mp4`、`http:`、`file:`、`data:` 等非 `mediaobj://` 媒体源都会抛 `EDITOR_RENDER_INVALID_MEDIA_SOURCE`。

### Critical 2: watchdog 清理 DONE + output

- `releaseEditorRenderLockForTask` 改为只按 `renderTaskId` 匹配 editor project，不再要求 `renderStatus='PROCESSING'`。
- terminal/watchdog 失败补偿现在会把同一 `renderTaskId` 的 editor project（包括已写成 `DONE` 的情况）回滚到 `FAILED` 并清空 `renderOutputMediaObjectId`。
- `IDLE` 取消路径仍只清 `renderTaskId`，不影响非 editor task；补偿范围由 `renderTaskId` 限定在 editor project 表。

### Important: smart_crop 真硬闸

- worker 的 `actualSeconds` 改为从当前加载的 `projectData` 中选中 video element 的 `e - s` 计算并 `ceil`，不再使用任务 payload 里的旧 `durationSeconds`。
- `frozenSeconds` 来自 payload 预冻结秒数（缺失时才回退当前片段），持久化 timeline 前比较 `actual <= frozen`；超出抛 `ENHANCE_BILLING_FREEZE_UNDERESTIMATED`，不写 timeline、不结算少量旧时长。
- CAS retry 过程中每次基于最新 projectData rebuild 后也重新执行硬闸，防止版本冲突后加载到更长片段仍被写入。

### 对其他任务影响

- `task/service.ts` 的变更只更新 `NovelPromotionEditorProject` 且以 `renderTaskId` 为条件；非 editor task 没有对应记录，不会被清理。
- 媒体白名单只应用在 Twick 媒体元素/props 资源字段，普通字幕文本和 metadata 不受影响。
